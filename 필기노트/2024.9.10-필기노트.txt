====================
※ 상속 : Inheritance ※
1. 정의
다른 클래스가 가지고 있는 멤버(필드, 메소드)들을 새로 작성할 클래스에서 직접 만들지 않고
상속을 받음으로써 새 클래스가 자신의 멤버처럼 사용할 수 있는 기능
--> 다른 클래스 : 부모 객체
--> 새로 작성할 클래스 : 자식 객체
1) 목적 : 클래스의 재사용, 연관된 일련의 클래스들에 대한 공통적인 규약 정의
2) 장점
    (1) 보다 적은 양의 코드로 새로운 클래스 작성 가능
    (2) 코드를 공통적으로 관리하기 때문에 고드의 추가 및 변경 용이
    (3) 코드의 중복을 제거하여 프로그램의 생산성과 유지보수에 크게 기여
!! import : 외부 설계도를 수입해오는 것
!! 상속  : 부모로부터 물려받은 필드, 메소드를 마치 자신의 것처럼 사용하는 것


2. 상속의 사용
1) 방법
클래스 간의 상속시에는 extends 키워드 사용
2) 표현식
[접근제한자] class 클래스명(자식) extends 클래스명(부모){}
public class Academy extends Company{}

3. 단일 상속과 다중 상속
1) 단일 상속
: 클래스간의 관계가 다중 상속보다 명확하고 신뢰성 있는 코드 작성
  !! 자바에서는 다중 상속 미지원(==단일상속만 지원)

4. 상속의 특징
1) 모든 클래스는 Object클래스의 후손
2) 부모클래스의 생성자, 초기화 블록은 상속 안 됨
3) 부모의 private멤버는 상속은 되지만 직접 접근 불가

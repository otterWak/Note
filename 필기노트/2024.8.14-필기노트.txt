※시맨틱 태그※
- 태그 이름 자체가 해당 요소의 의미와 역할을 명확히 나타내는 태그

- div, span 기존 영역 분할에 주로 사용되던 태그는
  태그 이름만 봤을 때 영역 분할 역할 이외의 의미를 파악하기 어려움

- SEO(Search Engine Optimization) : 웹 접근성을 향상하는데 도움이 됨
  검색 엔진(검색을 하면 검색 엔진으로 내용이 들어감)
  크롤링(정보 수집 하는 행위)
  *크롤링 하는 이유 : 색인(index) 하기 위함

[제공하는 태그]

header 태그 : 문서의 제목, 머리말, 로고, 사이트 이름, 탐색 메뉴 등
                 작성 가능한 영역
footer 태그 : 문서의 하단 부분, 꼬리말, 정보(저작권 정보, 저자 정보) 작성 영역
nav 태그 : 네비게이션 역할(나침반 역할 : 다른 페이지, 사이트 이동)
             링크 작성 영역
main 태그 : 한 페이지에 한 번만 사용해야 하며, 문서의 핵심 콘텐츠를 감싸는 역할
section 태그 : 문서의 논리적인 섹션을 의미, 콘텐츠를 그룹화할 때 사용함
article 태그 : 본문과 독립된 콘텐츠를 작성하는 영역
                    (블로그 포스트, 뉴스 기사, 사용자 리뷰 ..)
aside 태그 : 사이드바(보통 양쪽), 광고 영역

- 시맨틱 태그는 div와 같은 block 형식


==반응형 태그 작성시==
@(css at-rule)
: 특정 조건이나 규칙에 따라서 스타일을 적용할 수 있게 해주는 css 명령어
@media(미디어 쿼리)
: 특정 조건(ex 화면 크기)에 따라서 다른 스타일을 적용할 수 있게 해줌
